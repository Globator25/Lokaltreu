# UAT-Plan – Schritt 12

## Ziel (Roadmap Schritt 12)
- **US‑1:** Admin registriert sich, pflegt Stammdaten und legt die erste Kampagne in < 5 Minuten ohne Hilfe an.
- **Mitarbeiter-UI:** Oberfläche mit exakt zwei Aktionen (Stempel vergeben, Prämie einlösen) ist ohne Schulung bedienbar.

## Teilnehmer & Pseudonymisierung
- 3–5 reale Zielnutzer:innen (Friseur, Kosmetik, Nagelstudio).
- Identifikation ausschließlich über IDs `UAT-01` … `UAT-05`.
- Keine Erfassung von Klarnamen, Kontaktdaten oder PII.

## Setup
- **Geräte:** je nach Teilnehmer iOS/Android Smartphone + Tablet/Laptop für Admin-Dashboard.
- **Browser:** Aktuelle Version von Safari/Chrome/Edge.
- **Ort:** Remote-Call oder vor Ort im Geschäft (Dokumentation anonym).
- **Prototyp:** Figma/Mock-Lokaltreu Prototyp, Version `step12-v1` (Link im Moderatoren-Deck, nicht öffentlich teilen).

## Messpunkte pro Aufgabe
- Zeit bis Abschluss (mm:ss)
- Hilfe benötigt (Ja/Nein)
- PASS/FAIL (Definition of Done erreicht?)
- Abbruchgrund (falls vorhanden)
- Verständlichkeit (Skala 1–5)
- Fehlerverständnis (konnte Teilnehmer erklären, was schiefging?)

## Output-Artefakte
- UAT-Protokolle je Session (`UAT-Session-01..05`)
- Konsolidierte Findings + Severity (`Findings.md`)
- Priorisierten Maßnahmenplan (`Priorities.md`)
- UX-Entscheidungen mit Schema-Impact (`UX-Decisions-Schema-Impact.md`)
